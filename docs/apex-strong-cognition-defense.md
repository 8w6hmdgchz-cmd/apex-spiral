# 强认知防御检查清单

## 触发条件
以下情况必须执行防御检查:
- 事实性问题 (历史/科学/技术细节)
- 用户要求"确定"、"准确"
- 需要给出结论的任务

## 三问流程

### 1. 置信度自评
- 我对这个答案有多大把握?
- **< 80% 把握 → 必须说"不确定"或"可能"**

### 2. 证据链
- 最小证据链是什么?
- 能否说出来 (不要只在心里想)?

### 3. 反例
- 有什么反例可以推翻它?
- 想到了吗?

## 置信度降级规则

| 情况 | 置信度调整 |
|------|----------|
| 单路径推理 | -0.2 |
| 无外部证据 | -0.3 |
| 无法列出反例 | -0.15 |
| 用户要求"确定" | 必须 ≥0.8 否则说"不确定" |

## 多路径强制

遇到以下情况，必须多路径验证:
- 医学/药物/靶点相关问题
- 涉及实验结果/数据
- 用户明确要"准确"

## 输出模板

当置信度 < 0.8 时:
```
根据当前证据，我的结论是 [X]。
置信度: Y% (因为 [原因1]、[原因2])
如有需要，我可以进一步验证 [具体方向]。
```

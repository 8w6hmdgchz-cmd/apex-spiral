## 2. 发现的 bug

### Bug 1：dg 平台化
- 现象：`dg` 长时间固定在 `0.590764`
- 含义：增长指标对真实变化不敏感，进入“重复输出 = 持续成长”的假稳态

### Bug 2：phi 分辨率过低
- 现象：近期 `phi` 基本只在 `7.5 / 8.2` 两档间切换
- 含义：当前 `phi` 过度依赖环境压力等粗粒度信号，难以反映任务级变化

### Bug 3：缺陷历史重复，未形成新诊断
- 现象：91–95 轮 defect summary 基本重复
- 含义：系统还在报错，但报的是同一类老错，说明缺陷识别模块没有继续细化

### Bug 4：修复记录“恒成功”
- 现象：`repair_history.jsonl` 连续大量轮次固定 `amount=0.400 success=true`
- 含义：修复动作像模板回填，不像真实验证后的结果；有“声明修复即算修复”的倾向

### Bug 5：21354 / 12354 切换存在，但收益比较缺失
- 现象：模式持续交替，但 `phi/dg` 差异几乎没有体现出可解释收益
- 含义：模式切换更像排班，不像实验；没有把两种流程当 A/B 测试来学

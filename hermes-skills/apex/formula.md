# APEX 公式代入自检

## When to Use
- 收到任何任务/问题/请求时
- 发现错误或 bug 时
- 公式/工具/系统更新时

## Procedure

### 3秒自检（必须脱口而出）

**问题1：代入自己(2)**
- 这个任务需要什么能力？我有吗？差距多少？

**问题2：代入公式(1)**
- 用 Φ_APEX 的 Ψ/∇/Ξ/Γ 四要素照镜子
- 我自己有没有这个问题？

**问题3：举一反三(5)**
- 我之前有类似经验吗？
- claim 是否混淆了？

**问题4：查记忆(3)**
- 相关信息在记忆中吗？
- 需要巩固吗？

**问题5：选择路由(4)**
- REPAIR / OPTIMIZE / EXPLORE / INNOVATE / CURATE？

## APEX 主公式

```
ΔG = (Λ × Θ × K × ξ × Ψ × Φ) / (H × T × ε)
```

| 维度 | 含义 | 当前目标 |
|------|------|---------|
| Λ_root | 根能力 | >0.85 |
| Θ_llm | LLM推理 | >0.85 |
| K_master | 主导性 | >0.80 |
| ξ_anti | 幻觉防御 | >0.75 |
| Ψ_host | 健康稳态 | >0.85 |
| Φ_positive | 正反馈 | >0.80 |
| H_entropy | 信息熵 | <0.60 |
| T_cycle | 迭代周期 | <2.0 |
| ε_repair | 自修复率 | >0.80 |

## Pitfalls

- **跳过自检**：不要等别人提醒，每次任务强制执行
- **只看外部**：先代入自己，再看外部系统
- **claim混淆**：把"可能的结论"当"已验证的事实"

## Verification

执行后确认：
1. 能说出来"代入自己(2)"的答案
2. 能说出来"代入公式(1)"的答案
3. 能说出来"举一反三(5)"的答案

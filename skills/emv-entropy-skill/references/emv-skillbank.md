# Emv Entropy SkillBank v1

## long_context_distill
- Trigger: input is a long document, paper, transcript, or design dump.
- Procedure: extract claims/procedures/formulas/constraints; cluster into candidate skills; remove one-off details.
- Verification: each skill must solve at least one held-out question generated by Challenger.
- Failure recovery: if skill is too broad, split by trigger; if too narrow, merge with adjacent repeated pattern.

## challenger_task_generation
- Trigger: need to test whether a skill generalizes.
- Procedure: generate tasks across easy/medium/hard and adversarial variants; include at least one task that requires combining two skills.
- Verification: task must have an expected evidence path or deterministic scoring rubric.
- Failure recovery: discard tasks that are impossible, ambiguous, or only test memorization.

## judge_grounded_scoring
- Trigger: Reasoner output needs evaluation.
- Procedure: score correctness, evidence grounding, transferability, cost, and robustness from 0..1.
- Verification: low evidence or unsupported claims cap score at 0.6.
- Failure recovery: require citation/local evidence or mark unknown.

## replay_stability
- Trigger: repeated mutation or adversarial self-play causes skill drift.
- Procedure: mix latest tasks with older trajectory set; require no major drop on older tasks before accepting mutation.
- Verification: accepted mutation must keep replay score above baseline threshold.
- Failure recovery: rollback mutation or keep both old and new variants with narrower triggers.

## gini_entropy_mutation_select
- Trigger: multiple possible skill mutations exist.
- Procedure: compute Gini impurity, entropy, Gini gain, information gain from task outcome partitions; choose highest validated gain.
- Verification: selected mutation improves held-out or replay score, not just training score.
- Failure recovery: if gains are tied or noisy, choose simpler/readable mutation.

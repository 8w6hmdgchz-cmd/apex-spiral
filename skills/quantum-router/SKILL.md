# 量子路由 · 高维推理系统

## 核心理念

**叠加态并行**: 同时探索多条推理路径，通过概率波函数收敛最优解

## 架构

```
用户请求
    ↓
┌─────────────────────────────────────┐
│         量子叠加态 ( superposition )   │
│    同时生成 N 个平行推理分支          │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│         量子纠缠 ( entanglement )     │
│    分支间信息共享与状态同步           │
└────────────┬────────────────────────┘
             ↓
┌─────────────────────────────────────┐
│         量子测量 ( measurement )      │
│    概率幅收敛 → 选定最优路径          │
└────────────┬────────────────────────┘
             ↓
         响应
```

## 路由策略

### 叠加态分支数
- **简单任务**: 2-3 个分支
- **复杂推理**: 5-7 个分支  
- **关键决策**: 9+ 个分支

### 分支类型
1. **逻辑链** - 线性推理
2. **发散链** - 头脑风暴
3. **逆向链** - 从目标倒推
4. **类比链** - 相似问题映射
5. **元认知链** - 自我审视

### 收敛算法
```
1. 每个分支计算置信度 P(i)
2. 信息增益 G(i) = -log2(P(i))
3. 期望价值 E(i) = P(i) × G(i)
4. 选择 E(i) 最高的分支
5. 或合并多分支结果
```

## 与河图洛书集成

```python
# 量子路由调用
from hetu_luoshu import HetuRouter

router = HetuRouter()

# 叠加态分发
branches = router.quantum_split(
    task,
    branch_count=5,
    strategy=["logic", "divergent", "reverse", "analogy", "metacog"]
)

# 并行执行
results = router.parallel_execute(branches)

# 量子收敛
final = router.quantum_collapse(results, method="probability")
```
